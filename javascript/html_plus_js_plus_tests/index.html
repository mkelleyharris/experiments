Demo HTML + JS + Unit Tests

HTML and unit tests both try to use same code.js.

Old approach tried to bring in code.js via script tag as text/javascript, and used Document.write. Result: If the code.js exports js functions/class, it works in test, but not html. Without exports, it works in html but not in tests. What's the solution to work in both HTML and tests?

Newer approach uses import within script tags in index.html and Window.divid.innerHTML. Now works in both tests and html. (Requires web server locally, not just file open. e.g. python3 -m http.server 9000) See text added dynamically below.
